1. Présentation du projet
Nom du projet : TSELEM – Chaque Pixel Compte
Objectif : Créer une application web complète vitrine + prise de rendez-vous, avec gestion de contenu (blog + témoignages).
________________________________________
2. Stack technique recommandée
Comme Replit ne supporte pas tous les frameworks avec bases lourdes ni l’envoi SMTP direct sans configuration spécifique :
Élément	Recommandation
Base de données	Mysql / PostgreSQL 
Authentification	JWT ou Passport.js / Flask-Login + OAuth2 Google
Emails	Google SMTP (smtp.gmail.com) avec compte client
Déploiement	Sur Replit avec URL publique .replit.dev
Stockage images	Dossier /static/uploads ou service externe (Cloudinary, Firebase Storage)

🗓️ Prendre rendez-vous
Fonctionnalités :
•	Sélection de type de séance (photo / vidéo / autre)
•	Choix de la date + créneau (entre 10h et 18h)
•	Durée : 1h / 2h / 3h
•	Coordonnées client (nom, email, téléphone)
•	Commentaire libre
•	Confirmation instantanée à l’écran
•	Envoi d’un email de confirmation (au client + TSELEM)
Détails techniques :
•	Créneaux enregistrés en DB 
•	Vérification d’indisponibilité avant validation
•	Envoi d’email via Google SMTP
•	Host: smtp.gmail.com  
•	Port: 465 (SSL) ou 587 (TLS)  
•	Auth: Email + App Password  
•	(Option) Générer un fichier .ics pour ajout au calendrier.

o	Jours ouvrables, horaires, infos contact.
________________________________________
8. Emails
8.1 Configuration SMTP Google
•	Serveur : smtp.gmail.com
•	Port : 465 (SSL) ou 587 (TLS)
•	Auth : adresse email TSELEM + mot de passe d’application (non le mot de passe principal)
•	Envoi d’emails :
o	contact@tselem.studio → vers client / admin.
o	Formats HTML simples (signature, logo, couleurs TSELEM).
8.2 Types d’emails
1.	Confirmation de rendez-vous (client)
2.	Notification de nouvelle réservation (admin)
3.	Accusé de réception contact
4.	Réinitialisation de mot de passe (admin)

9. Base de données – Modèle minimal
Table	Champs principaux
users	id, name, email, password_hash, role, google_id
appointments	id, client_name, email, phone, service_type, date, start_time, duration, comment, status
reviews	id, name, rating, message, project_type, is_validated
articles	id, title, slug, content, image_url, author_id, created_at, updated_at
